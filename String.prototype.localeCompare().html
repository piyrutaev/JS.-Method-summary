Справка по String.prototype.localeCompare()

Метод localeCompare() сравнивает текущую строку (this) с другой строкой с учётом локали и возвращает число, указывающее порядок следования строк.

Синтаксис

referenceStr.localeCompare(compareString, [locales], [options])

Параметры

  • compareString — строка для сравнения (обязательный параметр).
  • locales — строка или массив строк локалей, например: 'en', 'ru'.
  • options — объект с настройками:
    • sensitivity: "base", "accent", "case", "variant"
    • ignorePunctuation: true / false
    • numeric: true (сравнение чисел как чисел)
    • caseFirst: "upper", "lower", "false"

Возвращаемое значение

  • -1 — текущая строка идёт перед compareString
  • 0 — строки равны
  • 1 — текущая строка идёт после compareString

Примеры

'a'.localeCompare('b');                            // -1
'ä'.localeCompare('a', 'de');                      // 1 (немецкий алфавит)
'ä'.localeCompare('a', 'sv');                      // -1 (шведский алфавит)
'2'.localeCompare('10');                           // 1
'2'.localeCompare('10', undefined, { numeric: true }); // -1

Использование в сортировке

arr.sort((a, b) => a.localeCompare(b, 'ru', { sensitivity: 'base' }));

Значение и преимущества при сортировке

Метод localeCompare() особенно полезен при сортировке строк в массивах, так как:

  • Учитывает локаль: позволяет правильно сортировать строки в соответствии с языковыми правилами (например, в немецком и шведском языках буквы с умлаутами имеют разный порядок).
  • Гибкость: через опции можно контролировать чувствительность к регистру, акцентам и пунктуации.
  • Числовая сортировка: с опцией numeric: true строки вида "2" и "10" сортируются как числа, а не как строки.
  • Универсальность: метод работает кроссбраузерно и поддерживается во всех современных JavaScript-движках.

Это делает localeCompare() предпочтительным способом сортировки строк, особенно если вы работаете с многоязычными данными или пользовательским интерфейсом.
